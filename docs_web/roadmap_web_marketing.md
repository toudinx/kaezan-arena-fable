> **Como disparar:** *"rode o prompt MK-NN do `docs_web/roadmap_web_marketing.md`"*.
> **Lê:** README.md + roster_digest.md + o doc nomeado. **Escreve:** `docs_web/marketing/`.

## Princípios
- **Tom:** dark-fantasy premium (alvo Wuthering Waves), editorial e enxuto — bate com a identidade
  "Cathedral Ink + Aurum" do README. Nada de exagero de marketing barato.
- **PT como primário, EN opcional** quando o prompt pedir.
- **Verdade:** copy descreve o que o jogo realmente tem (puxe do README). Não prometer feature inexistente.

## Itens

### [ ] MK-01 — Descrição da loja / página inicial
- **Escreve:** `docs_web/marketing/store-description.md`
- **Objetivo:** pitch curto + descrição longa do jogo (gacha + roguelike de dungeon com alma de
  Tibia), seções de feature, 1 linha de cada pilar. Variações curta/média/longa.

### [ ] MK-02 — Patch notes a partir de um roadmap concluído (recorrente)
- **Escreve:** `docs_web/marketing/patchnotes-<versao>.md`
- **Objetivo:** transformar os itens `[x]` de um roadmap de implementação (que você apontar) em
  patch notes player-facing — destaques no topo, lista por categoria, tom de changelog premium.
- **Lê (exceção nomeada):** o `docs/roadmap_*.md` indicado (markdown, não código).

### [ ] MK-03 — Copy de banner / convocação (recorrente)
- **Escreve:** `docs_web/marketing/banner-<kaeli>.md`
- **Objetivo:** título do banner, subtítulo, 1 parágrafo de hype e CTA para o banner de uma Kaeli,
  usando a identidade/lore dela do digest.

### [ ] MK-04 — Captions curtas multiuso (recorrente)
- **Escreve:** `docs_web/marketing/captions-<tema>.md`
- **Objetivo:** lote de captions curtas (loja, push, social cruzado) por tema/Kaeli. PT/EN.
